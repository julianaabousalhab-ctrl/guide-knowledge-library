# 8.1 Writing Guide Comments — Scope Log

## Included

- Structuring comments as Reality → Lever → Next Step
- Using student-specific evidence such as quotes, interests, behaviors, and concrete moments
- Naming what the Guide has not yet delivered or unlocked
- Prioritizing one high-leverage growth area

These were included because they describe distinct writing behaviors supported across multiple Guide Comments.

## Not Included

- Grammar, spelling, and copyediting issues
- Comment length as a standalone BrainLift
- Repeating every score or subject shown elsewhere in the report
- General positivity without evidence
- Parent-response or parent-conversation practices

Length was treated as part of prioritization rather than a separate pattern. Parent interaction belongs under 8.2.

## Ambiguous

- Some comments used strong evidence but were too long.
- Some comments showed Guide ownership but lacked a concrete next step.
- Some comments contained several useful strategies tied to one subject.

These were classified by the primary behavior being evaluated. A comment could support one BrainLift while serving as an anti-pattern for another.

# 8.2 Giving Parent Feedback — Scope Log

## Included

- Describing observable behavior instead of character, motive, or fixed traits
- Explaining relevant context and then stating the Guide’s action

These were included because the reports provide direct evidence of how Guides frame written feedback for parents.

## Not Included

- Parent reactions, trust, satisfaction, or understanding
- Handling parent disagreement
- Follow-up calls or two-way conversations
- Whether parents acted on the feedback
- General comment structure already covered under 8.1

These were excluded because no parent-response data or Guide-parent conversation records were available.

## Ambiguous

- The node title suggests two-way feedback, but the available evidence is one-way, parent-facing writing.
- Some contextual explanations were accurate but risked sounding like excuses.
- Some negative wording was technically based on behavior but still inferred student intent.

The node was narrowed to written parent-facing feedback. Context counted as positive only when followed by a Guide-owned action.

# 8.3 Progress Communication — Scope Log

## Included

- Communicating trajectory through an earlier state, change or intervention, and current direction
- Ending with a measurable checkpoint that includes an action, measure, and timeframe

These were included because they are distinct: trajectory explains progress already observed, while a checkpoint defines how future progress will be evaluated.

## Not Included

- Static descriptions of current performance
- Generic claims such as “making progress” without a comparison point
- Future plans without a measurable target or review point
- Multi-session longitudinal patterns

Static status belongs under comment content rather than progress communication. Multi-session patterns were excluded because most reports covered only Session 1.

## Ambiguous

- Some comments described recent improvement without giving an exact baseline.
- Some plans included a target but no monitoring cadence.
- Some comments explained data anomalies without clearly showing a trajectory.

Evidence was included only when movement over time or a verifiable future checkpoint was explicit. Partial examples were treated as anti-patterns or left out.
