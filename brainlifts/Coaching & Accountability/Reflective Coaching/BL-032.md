# Observe the Student Performing the Task Before Diagnosing the Problem

## Pattern
Guides strengthen reflective coaching when they watch the student perform the actual task, inspect the exact work, or test the reported difficulty before deciding what is causing the problem.

Performance data and student descriptions are useful, but they may not reveal whether the barrier is:
  - content knowledg
  - strategy
  - task directions
  - technology
  - instructional fit
  - physical setup
  - the difference between completing an application and understanding the underlying concept

The recurring sequence is:

Identify the concern → observe a representative task → test what the student knows and does → diagnose the barrier → match the support to the evidence

The Guide does not need to observe every problem for an extended period. A short, purposeful sample may be enough when it directly tests competing explanations.

This pattern differs from diagnosing whether the barrier is strategy, instructional fit, or system failure. That pattern concerns the conclusion the Guide reaches. This pattern concerns the evidence-gathering method used before reaching that conclusion.
