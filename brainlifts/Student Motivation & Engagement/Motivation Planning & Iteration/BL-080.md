# Treat the Model as Scaffolding — Launch It Against a Diagnosed Gap and Design Its Retirement

## Pattern

A motivational model is temporary scaffolding for a specific gap, not permanent furniture. The strong practice has two faces:

1. **At launch — verify the gap.** A model is justified by a diagnosed shortfall ("we are not yet at 100% of students consistently meeting daily goals"), not by the general goodness of motivation. Where the base system already pulls — XP payouts, culture, pride in leveling up — the strong campuses explicitly decline to layer another incentive on top.
2. **At launch — design the exit.** The model's own description states what happens when the gap closes: expectations rise, goalposts step up toward a terminal state, or the support intentionally fades until the student runs the structure alone. The end state is the student or level performing without this scaffold.

The mechanism: incentives that exist "because motivation is good" have no off-ramp, and rewards that never retire train students to expect them. Models anchored to a gap can be judged (did the gap close?), stepped up, and dismantled — leaving the habit, not the dependency.

Related: BL-068's visibility evidence illustrates this dependence risk in miniature — the sight of the incentive carries the motivation, which is acceptable scaffolding only while the model has an off-ramp.

## Evidence

*1- A sunset clause written into the model itself*

> "This system is used because we are not yet at 100% of students consistently meeting daily goals. It reinforces consistency and visibility of effort until all students reliably meet the standard, at which point expectations can be raised."

The same campus's raffle model carries the same logic: "This system is implemented because we are not yet seeing consistent, day-to-day demonstration of community-upholding behaviors."

## Anti-Pattern

The anti-pattern is running perpetual extrinsic models with no gap statement and no fade — until the reward becomes the precondition for any effort at all.

*Documented reward dependence*

> "Gia does not like to do anything without a physical reward wired in. If there is no game, alphas, or opportunity she can truly see in the moment, she has a difficult time meeting the challenge with the same amount of motivation. I would like to cultivate internal motivation of doing things for her future self."

The Guide's own next step names the missing half of the practice — cultivation of motivation that survives the reward's absence — after sessions of continuously running reward models for this student.

## Reusable Coaching Move

Use the Gap → Spec → Exit → Review process for any new model.

### 1. Gap: name what is broken

Write the specific shortfall the model addresses, with its current number ("we are not yet at X"). If you cannot name a gap the base system isn't already closing, don't build the model — strengthen recognition of what's working instead.

### 2. Spec: tie the reward to the gap

Design the earning condition so closing the gap is the only way to earn (see BL-066 for choosing the reward, BL-068 for making it visible).

### 3. Exit: write the retirement clause at launch

State in the model's documentation what happens when the gap closes: expectations rise, the goalpost steps up, the support fades, or the model retires. For individual models, name the end state — the student running the structure alone.

### 4. Review against the gap, not the enthusiasm

At each review (see BL-069 for moving the bar), ask whether the gap is closing. A beloved model whose gap has closed is due for graduation, not renewal.

The coaching standard is not:

> "The model is popular and students are motivated, so we keep it running."

The standard is:

> "We can say what gap this model exists to close, what the number is now, and what will replace the model when the gap is gone."

## Confidence: High — 4 instance clusters incl. multiple campuses independently declining to add incentives, + documented reward dependence as the contrast.
